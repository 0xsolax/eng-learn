## 3. 方向结论

设计语言暂定为“明亮、明确、有触感的课时探索界面”。

- 学习端：课时路径、单任务聚焦、即时解释反馈、低信息密度。
- 管理端：版本流水线、覆盖率矩阵、发布阻断项、高可扫描密度。
- 共享部分：青绿色主色、珊瑚与阳光状态色、Aa 词块标记、统一字体和组件状态。
- 不采用：自然日连续打卡、红心惩罚、排行榜、虚拟货币、竞品品牌绿色和角色复制。

## 4. 调研覆盖

调研使用 27 个一手来源，包括：

- Duolingo 官方品牌指南、产品与学习设计文章。
- Khan Academy Kids 官方产品页与儿童测评复盘。
- Quizlet 官方学习流程与题型研究。
- W3C WCAG 2.2 和认知无障碍资料。
- Apple Human Interface Guidelines。
- IBM Carbon Data Table 指南。

调研结论和视觉方案已交叉检查，没有发现与当前课时制、发布版本不可变或前后工作台分离原则的冲突。

## 5. 视觉验收

已使用 Playwright 真实渲染审阅稿并检查以下视口：

- 桌面：1440x1000。
- 移动：390x844。

验收结果：

- 页面级横向溢出：无。
- 浏览器控制台错误：无。
- Lucide 图标：71 个正常渲染。
- 页面章节：6 个正常渲染。
- 可点击按钮：没有高度低于 40px 的按钮。
- 学习端关键控件：主要按钮和答案选项达到 48px 以上。
- 关键状态：选中、正确、错误、禁用、键盘聚焦、处理中、网络失败均有视觉样例。
- `prefers-reduced-motion`：已提供禁用或压缩动效的 CSS 规则。

## 6. 视觉审阅重点

进入页面编码前，需要确认以下四项：

1. 是否接受青绿色作为独立品牌主色。
2. 是否接受第一版使用 Aa 抽象词块，暂不制作具象品牌角色。
3. 是否接受学习端优先服务 7 至 12 岁手机和平板用户。
4. 是否接受管理端保持克制和高密度，不与学习端同等卡通化。

如果以上方向获得确认，下一步才能拆分页面实施计划和组件任务；当前设计工件不代表已批准进入编码。

## 7. 已知边界

- HTML 是视觉审阅工件，不连接真实 API，也不承担生产行为验收。
- 第一版不包含正式品牌命名、完整插画系统、音频资产和具象角色设计。
- 儿童可用性仍需在页面原型完成后安排目标年龄用户观察测试。
- 320px、768px、1024px 和 1280px 的完整页面验证应在生产 Vue 页面实现阶段执行。
