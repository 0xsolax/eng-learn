## 2. 设计结论

学习端不增加新题型和新页面，而是在现有单题工作区内明确三层语境的递进：

```text
短语接触（S0）
  -> 单词识别（S1）
  -> 主动回忆（S2）
  -> 基础句应用（S3）
  -> 扩展句重组（S4）
  -> 独立输出与对照（S5）
```

视觉命题是“一条逐渐变长的语境轨道”。短语、基础句和扩展句使用同一任务画布，不使用三个并列卡片。学习者每次只看到当前任务，顶部阶段轨道只负责说明位置，不允许前端跳题。

## 3. 内容与阶段

以 `apple` 为例：

| 阶段 | 任务 | 当前可见内容 | 设计目的 |
| --- | --- | --- | --- |
| S0 | 看词识义 | `apple`、苹果、`An apple` | 低负担建立词义和最短语境 |
| S1 | 多项选择 | 苹果、英文选项 | 先完成识别，不要求立即拼写 |
| S2 | 主动回忆 | 苹果、拼写输入框 | 从识别进入无提示提取 |
| S3 | 基础句填空 | `I eat an ____` | 将目标词放入短句结构 |
| S4 | 扩展句拼装 | `I / eat / an apple / every day` | 在更长语境中理解结构 |
| S5 | 独立输出 | 中文提示、空白输入区 | 先输出，再揭示扩展句对照 |

S5 在学习者提交草稿前不得显示参考句。对照后继续使用现有 0–3 分自评，不增加自动评分。

## 4. 视觉层级

1. 顶部只显示课时、完成进度和当前阶段，不展示全部答案信息。
2. 当前任务指令是工作区第一阅读层级，英文目标内容是第二视觉锚点。
3. 语境长度通过文本占用宽度自然增长，不为每一层使用不同装饰色。
4. 选择项、输入框和词块继续使用现有 48px 以上触控尺寸。
5. 反馈区必须同时显示状态文字和答案信息，不仅依赖颜色。

## 5. 响应式规则

- 375px 手机：单列任务画布；阶段轨道允许水平紧凑排列，但页面不得横向滚动。
- 768px 平板：任务工作区最大宽度 680px，操作区保持单列。
- 桌面：任务工作区最大宽度 720px，置于稳定画布中央，不扩大文本行长。
- 扩展句词块可自动换行，不能缩小到低于 48px 的可点击尺寸。

## 6. 动效与状态

- 阶段进入时只使用一次 160–220ms 的轻微上移和淡入。
- 选择项、词块和按钮沿用现有按压反馈，不增加庆祝遮罩。
- 阶段轨道切换只移动当前标记，不滚动或闪烁整页。
- `prefers-reduced-motion: reduce` 时取消位移动效和自动滚动。
- 加载、错误、提交中、已提交和恢复状态均沿用现有任务容器，不另开弹窗。

## 7. 验收重点

1. S0 只能看到短语，不能提前泄露基础句或扩展句。
2. S1 是选择题，S2 是主动输入，顺序不能倒挂。
3. S3 只使用基础句，S4 与 S5 使用扩展句。
4. S5 参考句在草稿提交前不出现在 DOM、接口响应或可访问名称中。
5. 375px 与桌面宽度下无横向溢出，键盘焦点顺序与视觉顺序一致。
6. 错答反馈、继续操作、刷新恢复和结课报告保持现有行为。
7. 浏览器控制台无错误，关键学习请求无 4xx/5xx。
