## 2. 设计结论

管理端采用“安静、高密度、可行动的内容工作台”。它与学生端共享品牌，不共享布局密度和情绪表达。

管理端首先解决五个问题：

1. 管理员能否安全进入。
2. 导入、构建、审阅和发布分别已经完成了哪些事实，还有哪些阻断。
3. 哪些问题正在阻止发布。
4. 下一步最应该执行哪个动作。
5. 操作失败、结果未知或会话失效后如何安全恢复。

设计不追求“大卡片、数据装饰和看起来像后台”，而追求更短的判断路径：页面标题说明对象，状态说明现状，阻断项说明原因，主动作说明下一步，表格承担主要信息。

本版本不新增独立概览首页。`/admin` 登录成功后直接进入词库工作台，避免在没有独立聚合数据和真实管理动作时增加一次中转点击。未来只有在概览能够稳定提供待审批数、发布阻断、可发布版本和异常课程等真实聚合数据时，才单独设计概览。

## 3. 用户、场景与设计目标

### 3.1 主要用户

- 第一版管理员：单一配置型管理员，界面显示终端初始化时配置的账号名称。
- 使用场景：在桌面浏览器中导入词表、检查内容、处理缺口、批准发布、创建课程和管理学习码。
- 主要设备：1280px 至 1440px 桌面；768px 至 1199px 可完成主要任务；480px 至 767px 保持核心流程可用但不作为批量编辑主环境；320px 至 479px 的业务页以只读和桌面提示为主。

### 3.2 核心任务顺序

1. 终端初始化管理员账号。
2. 管理员登录。
3. 导入词表并创建草稿版本。
4. 构建 S0 至 S5 练习项目。
5. 定位覆盖缺口并审核练习。
6. 发布不可变版本。
7. 为学习者创建课程并安全交付学习码。

### 3.3 设计目标

- 登录页与学生学习码入口明显不同，不让管理员误以为需要 10 位学习码。
- 登录后直接到达第一个可执行工作区，不经过无业务价值的欢迎页。
- 页面首屏能够识别当前对象、状态、阻断原因和主要动作。
- 长列表和矩阵可快速扫描，关键操作不隐藏在 Hover 中。
- 破坏性操作、结果未知、发布不可变和一次性学习码具有明确风险说明。
- 前端不保存管理员密码、静态令牌或权威业务状态。

## 4. 与学生端的关系

### 4.1 共享设计语言

| 设计要素 | 两端共享规则 |
| --- | --- |
| 品牌 | 青绿色主色、`Aa` 词块标记、`eng learn` 名称 |
| 字体 | 中文使用系统无衬线字体；短英文和品牌允许使用 Fredoka 回退链 |
| 状态 | 青绿表示成功或当前状态，珊瑚表示错误或阻断，黄色表示提醒，天蓝表示信息 |
| 组件 | 8px 常规圆角、可见焦点、Lucide 线性图标、文字与图标共同表达状态 |
| 可访问性 | 键盘可达、颜色不是唯一信息、支持 forced-colors 和 reduced-motion |

### 4.2 必须区分的部分

| 维度 | 学生端 | 管理端 |
| --- | --- | --- |
| 目标 | 降低儿童答题负担 | 提高内容管理判断和操作效率 |
| 布局 | 720px 以内单任务窄列 | 224px 侧栏与最大 1440px 工作区 |
| 控件 | 48px 以上触控目标，主要选项 56px | 40px 控件，表格行高 44px |
| 信息量 | 一屏一个主要任务 | 表格、矩阵、流水线和批量操作 |
| 反馈 | 明亮、有触感、即时解释 | 克制、紧凑、说明对象和处理动作 |
| 动效 | 允许轻量按压与反馈进入 | 仅保留 120 至 160ms 状态变化 |

管理端不得使用学生端的大号课时节点、实体按钮阴影、奖励色块或庆祝动效。学生端也不得继承管理端的密集表格和领域术语。

## 5. 信息架构与路由

| 路由 | 页面职责 | 主动作 |
| --- | --- | --- |
| `/admin/login` | 管理员账号登录、会话恢复和错误处理 | 登录管理台 |
| `/admin` | 认证后重定向 | 转到 `/admin/source-versions` |
| `/admin/source-versions` | 版本列表、CSV 预览和导入 | 导入词表 |
| `/admin/source-versions/:versionId` | 构建、覆盖率、阻断项、审批和发布 | 服务端当前允许的主要动作 |
| `/admin/source-versions/:versionId/exercises/:itemId` | 结构化编辑、批准、禁用和只读查看 | 保存或批准 |
| `/admin/courses` | 创建课程、展示一次性学习码、查看课程和轮换学习码 | 创建课程 |

第一版左侧导航只显示两个真实入口：

- 词库工作台。
- 课程工作台。

不显示没有真实页面和数据的概览、通知、设置、用户权限或统计入口。

路由实现必须把登录入口与业务工作台分开：

- Vue 的 `/admin/login` 是 `AdminGate` 的同级公开路由，不是受保护布局的子路由。
- Worker 的 `GET/HEAD /admin/login` 和 `POST /api/admin/auth/login` 必须在通用 Admin 鉴权之前处理。
- `POST /api/admin/auth/logout` 允许在 Cookie 缺失、过期或已撤销时幂等清除浏览器 Cookie，但仍必须校验 Origin。
- `/admin/source-versions`、版本详情、练习详情、课程工作台及其他 `/api/admin/*` 业务接口继续经过管理员会话鉴权。
- 未登录访问受保护页面时进入登录页；未登录访问业务 API 时继续返回 JSON 401，不返回登录 HTML。

## 6. 管理员登录页

### 6.1 页面定位

登录页是认证前的独立页面，不加载管理端侧栏、顶部栏、词库名称、课程数据或任何业务 API。管理员完成登录后，才加载管理端应用框架和权威业务状态。

### 6.2 视觉构成

- 页面背景：`--color-canvas`。
- 登录表面：白色、1px `--color-line` 边界、8px 圆角、低阴影。
- 宽度：桌面 400 至 440px；窄屏使用可用宽度减去 32px。
- 顶部：40px `Aa` 标记、`eng learn`、辅助文字“内容管理”。
- 标题：`管理员登录`，24px / 700。
- 说明：`使用已在终端初始化的管理员账号进入内容工作台。`
- 字段：账号、密码；高度 40px；账号使用 `autocomplete="username"`，密码使用 `autocomplete="current-password"`。
- 主动作：全宽 `登录管理台`。
- 次要入口：普通文字链接 `前往学生学习端`，指向 `/app`。

登录页不展示营销语、插画、数据摘要、默认测试密码、10 位学习码输入框或“免费注册”。

### 6.3 登录状态

| 状态 | 页面行为 | 文案原则 |
| --- | --- | --- |
| 检查会话 | 保留稳定登录表面，暂不挂载可编辑字段 | 正在检查管理员会话… |
| 默认 | 账号和密码可输入 | 不提前暴露账号是否存在 |
| 提交中 | 禁止重复提交，按钮显示处理中 | 正在登录… |
| 凭证错误 | 保留账号、清空密码、焦点回到密码 | 账号或密码不正确 |
| 临时冷却 | 禁止继续提交，显示服务端返回的可重试时间 | 尝试次数过多，请稍后重试 |
| 未初始化 | 不显示默认凭证 | 管理员登录尚未配置，请先在部署终端完成初始化 |
| 网络失败 | 保留账号，允许重试 | 无法连接服务器，请检查网络后重试 |
| 服务异常 | 不猜测认证结果 | 登录服务暂不可用，请稍后重试 |
| 会话过期 | 卸载业务壳，回到登录页 | 登录已过期，请重新登录 |
| 登录成功 | 只跳转已注册的受保护管理路由 | 不显示中间成功页 |

错误区域紧邻表单并使用 `role="alert"`。错误文案不得区分“账号不存在”和“密码错误”，不得显示 Worker 异常、数据库信息或密码校验细节。

### 6.4 退出与恢复

- 完整桌面在侧栏底部显示真实管理员名称和“退出”；侧栏收窄或移动布局时，将同一入口迁移到顶部栏，两处不同时显示。
- 退出处理中禁用重复点击。
- 成功退出要求服务端立即撤销会话并清除 Cookie；旧 Cookie 重放必须被拒绝。前端先卸载私有页面状态，再使用 replace 进入 `/admin/login`，显示“已安全退出”。
- Cookie 缺失、过期或已经撤销时，退出仍幂等清除浏览器 Cookie 并返回成功。
- 退出失败时不得假装会话已经失效；页面说明“退出失败，会话仍可能有效，请重试”。
- 会话过期后重新登录，只恢复原站内管理路由，不恢复旧页面缓存；页面必须重新读取服务端状态。
- 浏览器后退不得恢复已退出工作台；受保护路由重新进入时必须再次向服务端确认会话。

## 7. 管理端应用框架

### 7.1 桌面框架

- 左侧固定导航：224px，白色表面，右侧 1px 分隔线。
- 品牌区：`Aa`、`eng learn`、`内容管理`。
- 主导航：40px 高，18px 图标，当前项使用浅品牌背景和深品牌文字。
- 侧栏底部：管理员名称、`内容管理员` 标签和退出动作；不显示无功能的头像菜单。
- 顶部栏：64 至 72px，显示面包屑、当前页面或版本上下文。完整桌面侧栏存在时不重复显示身份和退出；侧栏收窄或移动布局时，将同一身份与退出入口迁移到顶部栏。
- 主内容：最大宽度 1440px，24px 内边距，页面级垂直间距 24px。
- 提供“跳到主要内容”入口。

### 7.2 页面标题区

页面标题区固定回答四件事：

1. 当前在哪个工作区。
2. 当前管理的对象是什么。
3. 当前对象处于什么状态。
4. 当前最重要的动作是什么。

标题使用 24px / 700；说明使用 14px；状态使用 12px 紧凑标签。列表页和课程页的右侧最多保留一个主动作和两个次动作。版本详情标题区只展示对象与状态，构建、发布和丢弃统一放入唯一命令条；练习详情的审核动作统一放入审核区。破坏性动作不与主动作使用相同视觉权重。

### 7.3 顶部上下文

- 词库列表：`词库工作台 / 词库版本`。
- 版本详情：`词库工作台 / {词库名} / v{版本号}`。
- 练习详情：`{词库名} / v{版本号} / {单词} / {阶段}`。
- 课程页：`课程工作台 / 学习者与课程`。

顶部上下文来自当前路由和服务端数据，不使用静态“管理工作台 / 内容构建与课程”。

## 8. 词库版本工作台

### 8.1 默认布局

已有版本时，版本表格是首屏主体；“导入词表”由页面标题右侧主按钮展开同页导入工作区。没有任何版本时，导入工作区默认展开，并用空状态说明第一步。

导入仍使用同页工作区，不改为多步骤弹窗。原因是 CSV 预览、字段错误和结果未知需要稳定占位，不能依赖短暂浮层。

### 8.2 导入工作区

- 第一行：新词库 / 同词库下一版本切换。
- 第二行：词库名称或目标词库、CSV 文件。
- 第三部分：预览摘要和前 5 个词。
- 错误按 CSV 行号、字段和原因列出。
- 表单底部只保留一个主动作“创建草稿版本”。
- 结果未知时冻结其他导入动作，只提供“安全重试同一次导入”或“重新读取服务端状态”。

### 8.3 版本表格

| 列 | 规则 |
| --- | --- |
| 词库 | 主文字，允许两行但不得截断到无法辨认 |
| 版本 | 使用等宽数字或 tabular-nums |
| 状态 | 草稿、已发布、已丢弃；文字、边界和颜色同时表达 |
| 词数 / 分组 | 数字右对齐 |
| 审核进度 | `已批准 / 全部`，不只显示百分比 |
| 时间 | 仅展示服务端真实存在的创建或发布时间 |
| 操作 | 一个明确的“查看详情”链接 |

表头固定在表格滚动容器顶部。页面不得为了表格包裹多层装饰卡片。

## 9. 版本构建与发布工作台

### 9.1 首屏顺序

1. 词库名称、版本号和状态。
2. 构建、发布和丢弃命令条。
3. 导入、构建、审阅、发布四步流水线。
4. 发布阻断摘要。
5. 覆盖率矩阵和待审批项目。

命令条在桌面滚动时允许粘性停留于顶部栏下方，但不能遮挡页面标题或键盘焦点。

### 9.2 流水线

四个阶段保持固定顺序：导入、构建、审阅、发布。

- 已完成：勾选图标、完成数量和简短说明。
- 阻断阶段：珊瑚边界、阻断数量和处理入口。
- 未开始：中性边界和说明。

流水线只展示服务端已经返回的事实数量、版本状态、`readyToPublish` 和阻断项，不由前端根据数量猜测唯一“当前阶段”。未来若需要高亮当前阶段，必须先由服务端提供明确的 `workflowStep` 或等价字段。流水线用于解释状态，不伪装成可任意跳转的步骤导航。

### 9.3 发布阻断项

阻断项优先显示可行动信息：

- 对象：哪个单词。
- 阶段：S0 至 S5 中哪一阶段。
- 题型：缺少或未批准的练习类型。
- 原因：内容缺失、校验失败、未批准或已禁用。
- 下一步：打开对应练习，或筛选全部同类缺口。

在 1280px 及以上，阻断摘要可位于覆盖率矩阵右侧；较窄视口放在矩阵之前。发布按钮被禁用时，附近必须有可读原因，不能只依赖禁用状态。

### 9.4 覆盖率矩阵

主视图采用“单词 × S0 至 S5”的矩阵，而不是把每个单词、阶段和题型拆成大量重复行。

- 每行一个单词；不为矩阵额外扩张当前 API DTO。
- S0 至 S5 每列显示已批准、草稿、缺失或禁用。
- 状态同时使用图标、短文字或可访问名称，不只使用颜色。
- 点击单元格进入对应练习项目；缺失状态只在服务端存在可创建或可修复路径时提供动作。
- 提供“只看缺口”筛选；筛选无结果时说明当前条件下没有缺口。
- 矩阵下方保留面向批量审批的紧凑列表，不重复展示无操作价值的完整内容。

批量审批必须保留现有高效和恢复语义：

- 只允许选择真实 `draft` 项目，并显示“已选数量 / 当前待审批总数”。
- “全选待审批”作用于当前权威待审批集合，不包含已批准、已禁用或缺失项目。
- 超过单次 500 项上限时由前端按稳定批次自动提交，并显示已确认数量和总数量。
- 某一后续批次失败时，明确显示已经成功的数量，随后重新读取服务端状态；不得盲目重放已经成功的批次。

### 9.5 发布与丢弃

- 发布确认显示词数、分组数和“发布后不可原地修改”。
- 丢弃确认说明只处理当前草稿，不影响已发布版本和已有课程。
- 确认使用页内稳定区域或可访问对话框，不在操作按钮附近临时跳动。
- 已发布版本移除编辑动作，显示“创建下一草稿版本”。

## 10. 练习项目审核页

### 10.1 桌面布局

- 左侧主区：结构化练习内容编辑，占可用宽度约三分之二。
- 右侧审核区：版本、单词、阶段、题型、当前状态、批准和禁用动作；允许在长表单中粘性停留。
- 已发布或归档版本保持同一布局，但所有字段只读，右侧说明如何创建下一版本。

### 10.2 编辑规则

- 按题型显示真实字段，不提供原始 JSON 编辑。
- 字段错误紧邻字段显示，同时在保存动作附近给出汇总。
- 页面存在未保存修改时，离开当前练习前必须明确确认。
- 保存期间禁用重复提交；并发状态变化时重新读取服务端，不能覆盖较新内容。
- 批量审批继续在版本页完成；单项审核页只处理当前项目。

连续上一项、下一项或“保存并批准下一项”只有在服务端提供稳定审核顺序后才实施。本设计不要求前端凭当前列表猜测下一项。

## 11. 课程工作台

### 11.1 默认布局

已有课程时，课程表格是主体；“创建课程”由标题区主按钮展开同页表单。没有课程时，创建表单默认展开。没有已发布版本时，表单不可提交，并提供前往词库工作台的明确入口。

### 11.2 创建课程

- 学习者姓名和已发布词库版本处于同一紧凑表单。
- 主动作使用“创建课程并生成学习码”。
- 不要求管理员填写学习进度、阶段或课时日期。
- 结果未知时只允许安全重试同一操作。

### 11.3 一次性学习码

- 学习码使用等宽数字或 tabular-nums，尺寸明显高于周围正文。
- 显示“仅本次显示”的安全提示。
- 提供明确的“复制学习码”按钮和复制成功/失败反馈。
- 关闭前使用“我已安全记录”，关闭后列表不得恢复学习码。
- 页面、日志、测试截图和错误产物不得长期保存学习码。

### 11.4 课程表格

| 列 | 规则 |
| --- | --- |
| 学习者 | 主文字 |
| 词库版本 | 名称与版本号 |
| 当前课时 | `第 N 课`，数字右对齐 |
| 状态 | 学习中、已暂停、已完成 |
| 操作 | 第一版只保留轮换学习码 |

轮换学习码的确认应出现在触发位置附近的可访问区域或对话框，不放在长表格末尾。确认文案必须说明旧会话将失效。

第一版不增加搜索、排序和多条件筛选；只有真实课程数量证明列表已难以扫描时再单独设计，不为假设规模提前增加控件。

## 12. 组件与视觉规格

### 12.1 尺寸与密度

| 项目 | 规格 |
| --- | --- |
| 页面标题 | 24px / 1.3 / 700 |
| 区域标题 | 18px / 1.4 / 650 |
| 正文 | 14px / 1.6 |
| 表格正文 | 13px / 1.45 |
| 辅助信息 | 12px / 1.4 / 600 |
| 输入框和按钮 | 高度 40px |
| 表格行 | 最小高度 44px |
| 状态标签 | 高度 24px |
| 页面间距 | 24px |
| 区域内间距 | 12px 或 16px |

现有页面中大于 24px 的管理端标题和 52px 的普通课程表格行，应在实施时统一回到本规范；一次性学习码、确认区或需要两行内容的特殊行除外。

### 12.2 表面与深度

- 页面背景使用 `--color-canvas`。
- 数据区使用白色表面和 1px 分隔线。
- 常规容器圆角不超过 8px。
- 表格、命令条和流水线主要依靠边界和背景明度分层，不使用大阴影。
- 只有登录表面、确认对话框和浮动菜单允许使用阴影。

### 12.3 状态语义

| 状态 | 视觉语义 |
| --- | --- |
| 当前 / 可继续 | 品牌浅底、深品牌文字 |
| 成功 / 已发布 | 品牌色图标与文字，配清晰状态名称 |
| 提醒 / 草稿 | 黄色浅底、深色文字 |
| 错误 / 阻断 | 珊瑚浅底、深珊瑚文字和警示图标 |
| 已禁用 / 已丢弃 | 中性底、边界和明确文字 |
| 信息 | 天蓝浅底和信息图标 |

成功、错误和处理中反馈优先放在动作附近的紧凑通知区。发布阻断、结果未知、一次性学习码和会话失效仍使用稳定的大型状态区域，不使用自动消失 Toast。

## 13. 交互、防错与恢复

- 一个页面区域只设置一个主要动作。
- 管理写操作等待服务端确认后再更新权威状态，不用前端乐观发布、批准或轮换学习码。
- 网络结果未知时保留原 operation token，只允许安全重试或重新读取权威状态。
- 表单错误保留可修复输入；密码错误除外，密码字段清空。
- Loading 不先显示最终 Empty；Error 不显示旧缓存冒充当前状态。
- Published 状态必须移除写操作，不能只禁用输入框。
- 危险操作说明影响对象、不可逆部分和不受影响的对象。
- 关键动作不能只放入图标菜单；每行主动作不超过两个。
- 键盘焦点在确认、取消、失败和路由返回后落到仍可操作的明确目标。

## 14. 身份与安全体验边界

- `/admin/login` 允许未认证加载，但不得请求或显示业务数据。
- 未登录访问管理页面时进入登录页；管理 API 继续返回 JSON 401，不返回登录 HTML。
- 登录、退出和所有基于浏览器身份的管理写请求必须校验 Origin；受控 service token 调用不依赖浏览器 Origin，其安全边界是独立静态令牌和 API-only 路由。
- 浏览器只持有 Secure、HttpOnly、SameSite 管理员会话 Cookie，不持有管理员密码、密码校验值或 `ADMIN_API_TOKEN`。
- 登录成功的返回路径只接受路由表中已注册且受保护的管理路由，明确排除 `/admin/login`；空值、未知值和非法值统一进入 `/admin/source-versions`，禁止开放重定向和登录页自循环。
- 失败次数限制由服务端决定；UI 只展示服务端提供的可重试状态，不进行可绕过的本地计数。
- 不做永久账号锁定；唯一管理员的账号重置通过终端配置流程完成。
- 会话失效立即卸载业务壳，重新登录后完整读取服务端状态。
- 只有会话缺失、过期、撤销或身份无效等稳定错误码可以触发业务壳卸载和登录跳转。Origin 拒绝、凭证错误、临时冷却、配置未完成和服务异常保留当前页面上下文，显示对应可恢复状态，不能把所有 401/403 统一解释为会话失效。

配置型管理员是现有 Cloudflare Access 和服务令牌之外新增的第三种身份来源，三者不得互相覆盖：

1. 请求包含 Cloudflare Access assertion 时，必须验证 Access；无效 assertion 不回退到其他身份。
2. 没有 Access assertion 时，浏览器请求可以验证应用层管理员会话 Cookie。
3. `x-admin-token` 只服务受控 API 调用，不用于加载管理端页面、建立浏览器会话或充当登录密码。
4. 管理员会话 DTO 必须返回稳定 `source` 和服务端确定的显示名称；配置型账号使用终端初始化的账号名，Access 使用服务端认可的邮箱或主体显示名。

登录临时冷却目前没有现成持久状态支持，本设计只冻结可见状态，不把它冒充已经存在的安全能力。实施计划必须在 D1、Cloudflare 原生限流或其他跨 Worker isolate 的服务端机制中选择一种，冻结错误码、`retryAfter`、清理和并发规则；禁止使用单个 Worker 内存计数。

## 15. 响应式与无障碍

### 15.1 响应式

| 视口 | 规则 |
| --- | --- |
| 1280px 以上 | 224px 侧栏、完整导航文字、最大 1440px 工作区、矩阵与阻断双列 |
| 1200 至 1279px | 224px 侧栏、完整导航文字、主区与矩阵单列 |
| 768 至 1199px | 72px 图标侧栏、主区单列、详情审核区改为普通区块 |
| 480 至 767px | 顶部品牌和横向导航、表格容器横向滚动、显示桌面编辑建议 |
| 320 至 479px | 登录完整可用；管理端业务页以只读和使用桌面提示为主，不提供发布、丢弃、批量审批或轮换学习码等写操作 |

页面本身不得产生横向滚动；只有明确标记的表格或矩阵容器可以横向滚动。

本方案明确以普通区块替代 `DESIGN_0713` 在 768 至 1199px 详情页使用全高抽屉的建议。当前管理端已经使用独立详情路由，普通区块更少引入焦点、滚动和关闭状态，也符合第一版最小实现边界。

### 15.2 无障碍

- 普通文字对比度不低于 4.5:1；非文本控件和焦点指示不低于 3:1。
- 主要操作可通过键盘完成，焦点顺序与视觉顺序一致。
- 图标按钮具有 `aria-label` 和可见 tooltip。
- 表格使用正确的表头和行标题语义，数字列保持可扫描对齐。
- 状态使用文字、图标或形状和颜色共同表达。
- 登录错误、操作结果和一次性学习码使用适当的 live region，但避免重复播报整页。
- reduced-motion 下移除位移动画，不丢失任何状态。
- 200% 等效重排后登录和关键管理动作仍可访问。

## 16. 管理端文案规则

- 使用明确领域词：词库版本、练习项目、覆盖率、草稿、已发布、课程、学习码。
- 错误说明对象、原因和下一步，例如“4 个单词缺少 S4 句子拼装项目，补齐后才能发布”。
- 不使用“出错了”“操作失败”作为唯一信息。
- 不使用儿童端鼓励文案、夸张感叹号或庆祝语言。
- 不向管理员暴露数据库字段名、内部异常堆栈和认证实现细节。
- 学习码和管理员账号使用完全不同的名称，避免再次混淆入口。

## 17. 视觉与交互验收

### 17.1 必验页面与状态

1. 登录默认、错误、临时冷却、未初始化、网络失败、会话过期和已退出。
2. 词库空状态、已有版本、CSV 预览成功和校验失败。
3. 草稿未构建、存在阻断、可发布、发布确认和已发布只读。
4. 覆盖率完整矩阵、只看缺口和筛选无结果。
5. 练习编辑、字段错误、未保存离开、批准、禁用和只读。
6. 无已发布版本、创建课程、一次性学习码、复制结果和轮换确认。

### 17.2 视口

- 登录页：375×812、1280×800。
- 管理端：768×1024、1024×768、1280×800、1440×900。
- 等效重排：200%。

### 17.3 通过标准

- `/app` 明确是学习码入口，`/admin/login` 明确是管理员账号入口。
- 未认证时不显示管理端业务壳和任何业务数据。
- 登录后直接到达词库版本工作台。
- 1280px 下页面级无横向滚动，核心动作和阻断原因首屏可见。
- 表格标题固定，数字对齐，状态在灰度和 forced-colors 下仍可辨认。
- 版本存在缺口时不能把发布动作误表现为可用。
- 已发布版本不存在任何可成功执行的原地编辑动作。
- 一次性学习码关闭后不能从页面列表恢复。
- 会话过期、结果未知和网络失败均有可执行恢复路径。
- 页面没有通知、设置、概览等无真实能力的死入口。
- 前端 bundle、storage、日志和测试产物不包含管理员密码、密码校验值、静态管理员令牌或学习码。

## 18. 实施优先级与边界

设计确认后，实施应另写 `PLAN_`，按以下顺序进行：

1. 管理员终端配置、第三种应用会话身份、登录 API、会话 Cookie、服务端限流和安全错误契约。
2. `/admin/login`、会话恢复、退出和 `AdminGate` 路由调整。
3. 管理端顶部上下文、真实管理员信息和退出入口。
4. 词库导入区、课程创建区的按需展开和首屏密度收口。
5. 覆盖率矩阵、阻断摘要和表格固定表头。
6. 练习审核布局、一次性学习码复制和轮换确认位置优化。
7. 指定视口、键盘、会话、安全和真实 Worker/D1 浏览器验收。

本设计不授权编码、远端 Secret 写入、migration、部署或生产数据操作。登录会话时长、密码校验参数、失败冷却参数和 Cookie 生命周期应在实施计划中结合运行环境与安全测试冻结，UI 只依赖稳定状态和服务端返回的可重试信息。

## 19. 本轮冻结的设计决策

1. 管理端与学生端共享品牌语言，但使用明显不同的信息密度。
2. 管理员使用终端配置的账号密码登录，不使用 10 位学习码。
3. 登录页独立于管理端业务壳。
4. 第一版不新增独立概览首页，登录后直接进入词库工作台。
5. 第一版只保留词库工作台和课程工作台两个真实导航入口。
6. 管理端以流水线、阻断项、矩阵和表格为主要信息结构，不使用卡片墙。
7. 后台默认桌面高效操作，窄屏保持可访问但不作为批量编辑主环境。
8. 设计确认后先写实施计划，再进入 TDD 编码和远端发布授权。
