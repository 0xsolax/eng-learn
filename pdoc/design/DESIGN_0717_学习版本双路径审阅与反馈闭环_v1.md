## 2. 设计结论

学习版本构建完成后提供两条清晰路径：

```text
默认：进入审阅模式
  -> 使用学生同款题型体验
  -> 查看模拟判题
  -> 通过并下一题 / 反馈

次要：全部通过（N 项）
  -> 一次用户点击
  -> 内部按每批最多 500 项处理
  -> 权威重读版本状态
```

审阅模式是内容质检工作区，不是学生课程。它只复用题目渲染和纯判题语义，不创建课程、课时、任务、答题日志或学习状态。

## 3. 版本详情双路径

“审阅与批准”区域固定显示当前审阅摘要：待审阅、需重构、已批准和已禁用数量。

- `进入审阅模式` 为主动作，进入独立管理端审阅路由。
- `全部通过（N 项）` 为次动作，不显示逐项选择、全选或确认弹窗。
- 全部通过只在版本可修改、存在草稿、无待处理反馈、无禁用项且覆盖完整时启用。
- 超过 500 项时由前端稳定分批；后续批次失败不会伪造整体回滚，页面展示已确认数量并权威重读剩余草稿。

## 4. 审阅工作区

### 4.1 页面结构

1. 顶部显示词库名、版本号、当前位置和返回版本详情入口。
2. 状态摘要显示待审阅、需重构、已批准和已禁用数量。
3. 当前题显示单词、阶段、题型和派生审阅状态。
4. 中部复用六类 `TaskRenderer`，模拟输入与学生题型一致。
5. 底部提供 `反馈` 和 `通过并下一题`。
6. `上一题`、`下一题` 只导航，不改变持久化状态。

### 4.2 通过门禁

`通过并下一题` 只有在当前页面完成一次成功模拟判题后才启用。切题、内容更正、revision 变化或权威重读都会清空本地答案、S5 预览、判题结果和可通过状态。

模拟得分不自动批准题目。管理员必须显式点击通过，答错也可在了解判题反馈后作出人工决定。

### 4.3 完成态

全部项目批准后不再显示当前题，页面显示“全部审阅通过”，提供：

- 从头复查；
- 返回版本详情继续发布。

批准与发布保持两个独立动作。

## 5. 反馈与重构

`反馈` 在模拟答题前后均可打开。面板使用可聚焦的对话区域，支持 Escape 关闭，关闭后把焦点恢复到原触发按钮。

### 5.1 打回重构

- 反馈正文 trim 后必须为 1 至 2,000 字符。
- 字符计数和字段错误在输入区就地显示。
- 成功后题目变为 `draft + needs_rework`，反馈持久化，并自动进入下一待审阅题。
- 再次打回同一题会更新当前反馈，不创建历史线程。
- 有反馈的题目不显示通过按钮，旧批准、批量批准、禁用和发布入口也必须失败关闭。

“打回重构”不自动生成或覆盖练习，只把题目退回可编辑草稿。

### 5.2 直接更正

- 点击后按需读取包含答案的完整练习，只在反馈面板内显示现有结构化编辑器。
- 保存必须实际改变 prompt 或 answer；相同内容不能伪装成已更正。
- 成功后删除当前反馈、保持 `draft`，并留在当前题。
- 旧模拟结果立即失效，管理员必须使用更正后的题目重新体验和判题，才能通过。

## 6. 状态与持久化边界

练习主状态仍固定为：

```text
draft / approved / disabled
```

审阅状态由主状态和独立的当前反馈派生：

| 主状态 | 当前反馈 | 审阅状态 | 允许的主要动作 |
| --- | --- | --- | --- |
| draft | 无 | pending_review | 体验、更正、打回、批准 |
| draft | 有 | needs_rework | 查看或更新反馈、直接更正 |
| approved | 无 | approved | 复查、重新打回或更正 |
| disabled | 无 | disabled | 查看、重新打回或更正 |

数据库触发器禁止 `approved/disabled + 当前反馈`，并在题目内容实际变化时解决当前反馈。所有审阅决定使用 `content_revision` 比较并交换，每次成功决定只递增一次。

## 7. 并发与失败恢复

| 场景 | 页面行为 |
| --- | --- |
| 内容在另一窗口变化 | 权威重读，关闭基于旧 revision 的输入，要求重新判断 |
| 写请求失败但 revision 未变 | 保留反馈或结构化更正输入，允许原地重试 |
| 写响应丢失 | 重读项目状态与内容；只有 revision 恰好为请求期望值加一且结果匹配，才确认本次提交成功 |
| 写请求与权威重读均失败 | 保留未提交输入，锁定全部写入口，只允许重新读取服务端状态 |
| 版本被并发发布 | 立即移除陈旧题目与写入口，提示返回版本详情，不重放决定 |
| preview/evaluate 失败 | 保留本地输入，不启用批准，可原地重试 |

审阅窗口在组合读取前后复核版本状态和 revision；两次不一致时失败关闭。完整练习读取、preview、evaluate、三类审阅决定、原单项/批量批准和禁用也只使用同一份最新版本快照中的题目内容，不能把延迟单项读到的 prompt 或 answer 随新 revision 回写。

任何写失败都不会使用新 revision 自动重放管理员基于旧内容作出的决定。

## 8. 响应式与可访问性

- 479px 及以下只显示版本、进度、状态和“至少 480px”提示；不渲染表单、反馈、直接更正或批准入口。
- 480px 起显示完整交互。
- 1280×900 桌面下，审阅题目、反馈和更正保持单列阅读顺序，无横向溢出。
- 反馈面板具有对话语义、可聚焦、支持 Escape，并恢复触发按钮焦点。
- 操作结果使用状态或警告区域呈现，错误不只依赖颜色。
- 写操作进行中禁用重复触发；每个决定最多发出一次写请求，随后权威重读。

## 10. 非目标

- 不保存管理员模拟答案、得分或审阅会话。
- 不做多人会签、审核人身份、通知、评论线程或反馈历史。
- 不自动重新生成被打回的题目。
- 不修改学习阶段、题型、排课、错题回流或掌握度。
- 不允许原地修改已发布版本。
- 不增加学生端 API、AI、R2、语音、支付或多租户能力。
