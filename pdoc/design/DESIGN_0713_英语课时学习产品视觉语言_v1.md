## 2. 设计结论

第一版采用“明亮、明确、有触感的课时探索界面”作为统一设计语言。

学习端借鉴成熟学习产品的四类机制：

1. 用一条清晰路径告诉孩子现在学什么，不要求孩子自己规划复习顺序。
2. 每屏只处理一个主要任务，减少选择和阅读负担。
3. 答题后立即给出对错、正确答案和下一步动作。
4. 通过形状、颜色、进度和轻量动效提供成就感。

管理端不采用卡通化学习界面，而采用同一品牌色下的高密度工作台：

- 左侧稳定导航。
- 顶部展示当前词库版本和发布状态。
- 中部以“导入、构建、审阅、发布”流水线组织操作。
- 表格和覆盖率矩阵承担主要信息，不用装饰性卡片堆叠。

两端共享颜色、字体、图标、圆角、按钮按压反馈和状态语义，但使用不同的信息密度。

## 3. 本稿假设

本设计稿按以下使用条件制作：

| 项目 | 本稿采用的条件 | 设计影响 |
| --- | --- | --- |
| 主要学习者 | 7 至 12 岁儿童 | 中文指令短句化，主要控件不小于 48px |
| 学习设备 | 手机和平板优先，桌面可用 | 学习任务采用单列聚焦布局 |
| 管理者 | 家长或内容管理员 | 管理端以桌面 1280px 以上为主要工作区 |
| 单次学习 | 15 至 25 分钟 | 首页突出当前课，不展示复杂长期统计 |
| 品牌名称 | 暂用仓库名 `eng learn` | 不在本稿中决定正式中文品牌名 |
| 品牌角色 | 第一版使用抽象 `Aa` 词块标记 | 不复制任何竞品角色，也不承担插画生产成本 |

如果主要年龄下探至 6 岁以下，应重新评估文字量、拼写输入方式、语音引导和家长协助流程。本稿不自动覆盖该年龄段。

## 4. 竞品机制取舍

### 4.1 可以迁移的机制

| 来源 | 成熟机制 | eng-learn 转化方式 |
| --- | --- | --- |
| Duolingo | 线性学习路径降低“下一步学什么”的决策成本 | 用“第 N 课”路径展示当前课、已完成课和下一课 |
| Duolingo | 小课时、题型逐步变难、复习混入主路径 | 用 S0 至 S5 阶段和课时调度决定题型，不由前端自由选课 |
| Duolingo | 正确反馈及时、视觉状态明确 | 使用底部反馈区说明结果和正确答案，再允许继续 |
| Khan Academy Kids | 游戏、故事和角色用于降低儿童学习压力 | 第一版仅使用温和视觉和短指令，不增加独立游戏系统 |
| Quizlet Learn | 题型变化和针对薄弱内容的复习 | 由后端调度不同 `lessonTask`，UI 保持统一任务容器 |
| W3C / Apple | 一致导航、明确反馈、大点击区域、可减少动效 | 学习端 48px 触控目标，状态不只依赖颜色，支持 `prefers-reduced-motion` |

### 4.2 明确不复制的机制

- 不使用自然日连续打卡。项目调度单位是课时，日期不能影响复习节奏。
- 不使用红心耗尽、生命值或答错后阻断学习的惩罚机制。
- 不使用排行榜、联赛、虚拟货币和商店，它们不属于第一版训练闭环。
- 不复制 Duolingo 绿色、字体、猫头鹰角色、插画形状或按钮细节。
- 不把每个后台数据块都包装成大圆角卡片。
- 不用庆祝动效遮挡题目、解释或下一步按钮。
- 不用羞辱、催促或制造失去进度焦虑的文案。

## 5. 视觉主题与氛围

### 5.1 视觉主张

**学习端：明亮的课时探索。** 白色和浅青绿色作为主要环境色，珊瑚红和阳光黄只承担重点反馈。界面有明确触感，但不幼儿园化。

**管理端：安静的内容工作台。** 同一色板降低饱和度，以画布层级、细分隔线和紧凑表格建立秩序。首屏优先回答当前版本能否发布、缺什么、下一步做什么。

### 5.2 设计签名

统一视觉签名为“词块 + 课时路径”：

- `Aa` 词块作为产品标记、当前课节点和任务完成标记。
- 路径节点表示课时，不表示自然日。
- 词块允许使用不同状态色，但轮廓和几何比例保持一致。
- 第一版不引入具象品牌角色，避免插画体系在验证期成为额外依赖。

## 6. 色彩系统

生产实现使用 CSS 变量和 OKLCH；十六进制值用于设计工具和不支持 OKLCH 的回退。

| Token | OKLCH | Hex 回退 | 用途 |
| --- | --- | --- | --- |
| `--color-brand` | `oklch(0.62 0.13 183)` | `#13998D` | 路径节点、图标、选中状态 |
| `--color-brand-strong` | `oklch(0.50 0.105 183)` | `#0A746C` | 主按钮、深色品牌文字 |
| `--color-brand-soft` | `oklch(0.94 0.045 183)` | `#D7F4EF` | 选中背景、轻反馈背景 |
| `--color-coral` | `oklch(0.70 0.17 28)` | `#F56A5D` | 错误、需加强、关键提醒 |
| `--color-coral-soft` | `oklch(0.95 0.035 28)` | `#FDE5E1` | 错误说明背景 |
| `--color-sun` | `oklch(0.83 0.16 82)` | `#F4C64E` | 新词、奖励、当前目标 |
| `--color-sky` | `oklch(0.72 0.12 230)` | `#4FA7D9` | 信息提示、S3/S4 辅助状态 |
| `--color-ink` | `oklch(0.29 0.025 195)` | `#23343A` | 主要文字 |
| `--color-muted` | `oklch(0.51 0.025 195)` | `#60737A` | 次要文字 |
| `--color-canvas` | `oklch(0.975 0.008 183)` | `#F4F8F7` | 页面背景 |
| `--color-surface` | `oklch(1 0 0)` | `#FFFFFF` | 主要内容表面 |
| `--color-line` | `oklch(0.90 0.012 183)` | `#DDE8E5` | 分隔线和表格线 |

色彩约束：

- 页面约 60% 使用画布和白色，30% 使用文字和分隔层级，10% 使用品牌及状态色。
- 主按钮使用 `brand-strong` 配白色文字，不使用较浅的 `brand` 承担小字号文字背景。
- 错误状态同时显示图标、标题和说明，不只改变颜色。
- 学习端允许多色状态；管理端除状态标签外，以品牌青绿色和中性色为主。

关键文本组合的十六进制回退色对比度检查：

| 前景 / 背景 | 对比度 | 结论 |
| --- | ---: | --- |
| `#0A746C` / `#FFFFFF` | 5.63:1 | 主按钮与品牌文字通过 WCAG AA |
| `#23343A` / `#F4F8F7` | 12.08:1 | 主要正文通过 WCAG AA |
| `#60737A` / `#FFFFFF` | 4.96:1 | 次要正文通过 WCAG AA |
| `#A23F38` / `#FDE5E1` | 5.29:1 | 错误说明文字通过 WCAG AA |
| `#23343A` / `#F4C64E` | 8.03:1 | 阳光黄状态文字通过 WCAG AA |

## 7. 排版规则

### 7.1 字体策略

- 英文单词、课时数字和品牌标记：`Fredoka` 600 至 650。
- 中文界面和正文：`PingFang SC`, `Noto Sans SC`, system-ui, sans-serif。
- 管理端数字列启用 `font-variant-numeric: tabular-nums`。
- 正式实现应自托管字体或使用系统字体回退，不依赖在中国大陆不稳定的公共字体 CDN。
- 所有字距固定为 `0`，不压缩中文或英文字符间距。

## 8. 空间、圆角和深度

### 8.1 间距

统一间距级别：`4, 8, 12, 16, 24, 32, 48px`。

- 学习端页面水平内边距：手机 20px，平板 32px，桌面最大内容宽度 720px。
- 管理端侧栏宽度：224px；主内容最大宽度 1440px；内容间距 24px。
- 不为单个页面发明新的零散间距。

### 8.2 圆角

| Token | 值 | 用途 |
| --- | ---: | --- |
| `--radius-xs` | 4px | 标签、小型状态块 |
| `--radius-sm` | 8px | 卡片、输入框、按钮、表格容器 |
| `--radius-md` | 12px | 学习反馈底板、移动端底部面板 |
| `--radius-pill` | 999px | 进度条、筛选状态、圆形路径节点 |

卡片圆角不超过 8px。圆形和胶囊仅用于路径节点、进度和紧凑状态，不作为所有容器的默认形状。

### 8.3 深度

- 管理端主要依靠背景明度差和 1px 分隔线，不使用漂浮卡片墙。
- 学习端选项使用 `0 3px 0` 底部实体阴影形成触感。
- 按下时 `translateY(2px)` 并将底部阴影减为 `0 1px 0`。
- 弹出层只使用两级阴影：`0 8px 24px rgba(35, 52, 58, 0.12)` 和 `0 16px 48px rgba(35, 52, 58, 0.16)`。

## 9. 图标与视觉资产

- 产品图标统一使用 Lucide 线性图标，默认 20px，管理端 18px。
- 图标按钮必须有可见 tooltip 和 `aria-label`。
- 不混用 Lucide、Emoji、Font Awesome 或手绘图标。
- 第一版视觉资产以词块、路径节点、词义图片占位区为主。
- 没有正式图片时使用标明用途的占位区，不用低质量临时插画冒充生产资产。
- 后续若引入品牌角色，应单独建立角色造型、姿态、情绪和使用边界，不从竞品角色改色产生。

## 10. 动效与触觉反馈

### 10.1 统一动效

- 普通状态变化：120 至 160ms。
- 反馈面板出现：180ms，`opacity + translateY(8px)`。
- 页面内容进入：最多 220ms，不做连续弹跳。
- 只动画 `transform` 和 `opacity`。
- 所有交互遵循 `prefers-reduced-motion`，开启后移除位移动画。

### 10.2 标志性微交互

学习端的标志性微交互是“词块按压”：

1. 点击答案时选项向下移动 2px，模拟实体按键。
2. 正确后当前词块轮廓切换为品牌色，并显示简短解释。
3. 错误后不晃动、不扣生命值，显示珊瑚色说明和正确答案。
4. 反馈确认后再进入下一题，避免结果一闪而过。

管理端只保留按压缩放和 120ms 状态切换，不使用庆祝动效。

## 11. 学习端信息架构

```text
学习码进入
  -> 课程首页 / 当前课时路径
  -> 开始第 N 课
  -> 单任务答题
  -> 即时反馈
  -> 下一任务
  -> 错词回炉
  -> 课后报告
  -> 返回第 N+1 课入口
```

### 11.1 课程首页

首屏只回答四个问题：

1. 我现在是第几课。
2. 这节课有多少复习词和新词。
3. 预计需要多久。
4. 从哪里开始。

主要内容：

- 顶部课程名称和已完成课时数。
- 当前课大节点。
- 旧词复习、新词、预计时间摘要。
- 单一主按钮“开始第 8 课”或“继续第 8 课”。
- 向下可见前一课和下一课节点，让路径连续但不暴露复杂调度数据。

首页不显示：自然日连续打卡、排名、货币、商城、订阅推广和大段学习说明。

### 11.2 任务页

- 顶部只保留退出、进度和当前任务数量。
- 中部显示一条指令和一个主要题目区域。
- 选择题每个选项高度至少 56px。
- 固定底部动作区只出现一个主要动作。
- 提交前主按钮为“检查答案”，反馈后变为“继续”。
- 网络失败保留选择和输入，显示“重新提交”，不推进本地进度。

### 11.3 课后报告

报告先给结论，再给细节：

- 完成第 N 课。
- 本课正确率和完成任务数。
- 掌握更稳的词、需要再练的词。
- 下一课入口。

报告不展示儿童难以理解的 `easeFactor`、阶段计数或调度公式。

## 12. 管理端信息架构

```text
概览
词库版本
  -> 导入词表
  -> 预览与校验
练习构建
  -> 构建 S0-S5
  -> 覆盖率矩阵
  -> 单词练习详情
  -> 审核与发布
学习者与课程
  -> 创建学习者
  -> 绑定已发布版本
  -> 查看课程状态
```

### 12.1 工作台框架

- 左侧导航固定，当前栏目使用浅品牌背景和图标，不使用粗色条。
- 顶部区域显示当前页面、版本状态和主要动作。
- 主区采用全宽分段布局，表格是数据主视图。
- 右侧详情采用抽屉或独立详情页；内容构建主流程不依赖连续弹窗。

### 12.2 词库与构建页

页面首屏依次展示：

1. 词库名称、版本号和 `draft/published` 状态。
2. 导入、构建、审阅、发布四步进度。
3. 总词数、分组数、覆盖率、阻断项。
4. 覆盖率矩阵和筛选。
5. 当前最重要的下一步动作。

发布按钮必须在覆盖率未达标时禁用，并明确列出阻断原因。

## 13. 核心组件规范

### 13.1 按钮

| 类型 | 默认 | Hover | Active | Disabled |
| --- | --- | --- | --- | --- |
| 学习主按钮 | 深品牌底、白字、3px 底部深色层 | 明度轻微提升 | 下移 2px | 中性底、无底部层 |
| 管理主按钮 | 深品牌底、白字、8px 圆角 | 背景加深 | `scale(0.96)` | 中性底、文字降对比 |
| 次按钮 | 白底、1px 分隔线 | 品牌浅底 | `scale(0.96)` | 保留边界、降低对比 |
| 危险按钮 | 白底、珊瑚色文字 | 珊瑚浅底 | `scale(0.96)` | 不允许隐藏风险文案 |

### 13.2 输入框

- 高度：学习端 52px，管理端 40px。
- 默认 1px `line` 边界，聚焦使用 2px `brand` 焦点环。
- 错误说明紧邻字段，不能只在页面顶部汇总。
- 学习码按 4 至 6 位大字号分组显示，但不得强迫用户记忆后再进入另一页输入。

### 13.3 选项

- 整行可点击，至少 56px 高。
- 左侧可使用 A/B/C/D 序号，不用小型单选圆点作为唯一点击目标。
- 选中、正确、错误三个状态同时改变轮廓、图标和文字说明。
- 状态变化不得造成容器高度跳动。

### 13.4 状态标签

- 仅用于短状态，如“草稿”“已发布”“缺 4 项”。
- 高度 24px，圆角 999px，文字 12px。
- 不把普通命令放入胶囊按钮。

### 13.5 表格

- 文字左对齐，数字右对齐。
- 表头固定，行高 44px。
- 每行主动作不超过 2 个，其余进入菜单。
- S0 至 S5 覆盖率用图标和短标签表达，不只用六种颜色。

## 14. 状态与异常

每个关键页面必须设计以下状态：

- Loading：骨架或稳定的处理中状态，不显示最终空状态。
- Empty：说明为什么为空，并提供唯一下一步动作。
- Error：说明哪个动作失败、当前数据是否保留、如何重试。
- Unauthorized：管理端不展示业务壳，直接进入认证说明或 Cloudflare Access。
- Offline / Network retry：学习页保留当前答案，不推进任务。
- Incomplete lesson：重新进入时显示“继续第 N 课”，恢复未完成任务。
- Published immutable：已发布版本只读，编辑动作指向创建新版本。

## 15. 响应式规则

| 视口 | 学习端 | 管理端 |
| --- | --- | --- |
| 320 至 479px | 单列，20px 边距，底部固定动作区 | 不作为主要编辑环境，只提供只读或提示使用桌面 |
| 480 至 767px | 单列，最大宽度 520px | 导航折叠，表格允许横向滚动 |
| 768 至 1199px | 居中任务区，最大宽度 720px | 侧栏收窄为图标导航，详情使用全高抽屉 |
| 1200px 以上 | 课程首页可增加右侧课时摘要，任务仍保持窄列 | 224px 侧栏和全宽工作区 |

学习端所有关键控件最小 48x48px；管理端最小 40x40px。两端均不得依赖 Hover 才能发现关键操作。

## 16. 可访问性基线

- 正文文字与背景对比度不低于 4.5:1。
- 大字号文字和非文本控件不低于 3:1。
- 颜色不是状态的唯一信息来源。
- 页面结构使用正确的标题层级、按钮和链接语义。
- 提供可见键盘焦点和跳到主内容入口。
- 学习任务支持键盘选择和提交。
- 动效尊重系统减少动态效果设置。
- 不自动播放无法暂停的声音或动画。
- 中文和英文片段标记正确的 `lang`。
- 触控目标优先采用 48px，超过 WCAG 2.2 的 24px 最低要求，并满足 44px 增强目标方向。

## 17. 文案语气

### 学习端

- 指令一次只说一件事：“选出 apple 的中文意思”。
- 正确反馈具体：“答对了，apple 是苹果”。
- 错误反馈不羞辱：“再看一次，apple 是苹果”。
- 不使用“你怎么又错了”“连续学习要断了”等焦虑文案。
- 不用感叹号堆叠制造兴奋。

### 管理端

- 使用领域术语：词库版本、覆盖率、练习项目、发布阻断项。
- 错误文案说明对象和处理动作：“4 个单词缺少 S4 句子拼装项目，补齐后才能发布”。
- 不用“出错了”替代可行动的信息。

## 18. 实施边界

本设计方案只定义 UI 基调，不改变以下业务边界：

- 调度仍由后端按课时编号决定。
- 前端不计算阶段升级和下一次复习课时。
- 管理端只编辑草稿版本。
- 学习端只读取已发布内容和课时任务快照。
- 第一版不新增 AI、语音识别、R2、支付、排行榜或社交功能。

生产实现建议继续使用现有 Vue 3 和 scoped CSS，通过全局 CSS 变量提供 token。第一版不必为了视觉稿引入 Tailwind、UnoCSS 或完整组件库。

## 19. 视觉验收方式

### 19.1 视口

- 学习端：320x568、375x812、768x1024、1280x800。
- 管理端：1024x768、1280x800、1440x900。

### 19.2 必验页面

1. 学习码进入页。
2. 课程首页和当前课时路径。
3. 选择题、填空题、句子拼装和中文到英文任务。
4. 正确、错误、网络失败和恢复状态。
5. 课后报告。
6. 管理端词库列表、导入预览、覆盖率矩阵和发布阻断状态。

### 19.3 通过标准

- 孩子无需阅读说明即可找到当前课和开始按钮。
- 单任务页没有第二个竞争性的主操作。
- 375px 下按钮、选项和中文文案不溢出。
- 320px 下最长按钮文案可以完整显示或自然换行。
- 管理端 1280px 下核心表格无需页面级横向滚动。
- 所有状态在灰度截图中仍可区分。
- 开启减少动效后不丢失任何状态信息。
- 页面不出现竞品品牌资产、绿色主色复制或自然日打卡语义。

## 20. 本轮需要确认的设计决策

视觉评审重点确认以下四项：

1. 主色是否采用青绿色，而不是更接近 Duolingo 的亮绿色。
2. 第一版是否接受“抽象 Aa 词块”，暂不设计具象品牌角色。
3. 学习端是否以 7 至 12 岁和手机/平板为第一优先级。
4. 管理端是否接受明显更克制、更高密度的视觉，而不是与学习端同等卡通化。

确认后，页面实施应先建立 token 和两端应用框架，再按“课程首页、任务页、管理端内容流水线”的顺序完成首批页面。
