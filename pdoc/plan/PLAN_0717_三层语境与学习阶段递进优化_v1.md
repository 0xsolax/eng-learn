## 2. 实施结论

本次将新导入词库从“一个例句重复承担三个阶段”升级为“三层语境逐级展开”，同时保留六个学习阶段：

```text
S0  单词 + 释义 + 最短短语
S1  多项选择识别
S2  主动默写单词
S3  基础句填空
S4  扩展句拼装
S5  独立输出后对照扩展句
```

新 CSV 固定列为：

```text
word,meaning,examplePhrase,exampleSentence,exampleSentenceExtended,partOfSpeech
```

示例：

```text
apple,苹果,An apple,I eat an apple,I eat an apple every day,noun
```

不直接原地改变旧内容语义。词库版本持久化内容模型版本：历史版本按旧模型读取，新导入版本按递进模型构建。既有课程的题目快照继续使用创建当时的题型、提示和答案。

## 3. 目标与非目标

### 3.1 目标

1. 新导入词库必须同时提供短语、基础句和扩展句。
2. 新词库的 S0-S5 严格使用已确认的递进顺序和对应语境。
3. 旧已发布词库、旧草稿和既有 lesson task 快照继续可读取、作答和恢复。
4. CSV 模板、预览、管理端导入、服务端校验和 D1 持久化使用同一字段契约。
5. 发布覆盖检查按内容模型识别正确的阶段与题型组合。
6. 学习端六类任务继续通过现有统一渲染器展示，不引入新的题型数量。
7. 全量测试、构建、迁移和真实浏览器学习闭环通过后才完成验收。

### 3.2 非目标

- 不修改 S0-S5 的阶段数量。
- 不修改 StageEngine 的升降级、掌握度和跨课间隔公式。
- 不修改错题回流的 3-6 题间隔、每词同课最多 3 次或队列策略。
- 不修改 `partOfSpeech` 的用途和数据语义。
- 不原地重建已发布题目，不覆盖既有 lesson task 快照。
- 不增加 AI 生成、语音识别或自动纠错。
- 不用字符数或词数硬判定三层语境的长短关系；模板与界面提供填写指引，避免误拒合法表达。

## 4. 冻结的数据与阶段契约

### 4.1 内容模型

词库版本新增不可变内容模型标识：

```text
v1_single_sentence      历史单例句模型
v2_progressive_context  新三层语境模型
```

迁移默认值为 `v1_single_sentence`，因此所有迁移前数据自动保持旧语义。所有迁移后通过管理端新建或追加导入的版本写入 `v2_progressive_context`。

### 4.2 单词字段

新模型字段：

| 字段 | 必填 | 用途 |
| --- | --- | --- |
| `word` | 是 | 目标单词 |
| `meaning` | 是 | 中文释义或学习提示 |
| `examplePhrase` | 是 | S0 最短接触语境 |
| `exampleSentence` | 是 | S3 基础句填空 |
| `exampleSentenceExtended` | 是 | S4 拼装与 S5 对照 |
| `partOfSpeech` | 否 | 词性元数据 |

旧数据的新增语境列使用空字符串回填，但旧模型构建与读取不得依赖这些列。

### 4.3 阶段与题型映射

| 阶段 | v1 历史模型 | v2 递进模型 | v2 语境来源 |
| --- | --- | --- | --- |
| S0 | `recognize_meaning` | `recognize_meaning` | `examplePhrase` |
| S1 | `recall_word` | `multiple_choice` | `meaning` |
| S2 | `multiple_choice` | `recall_word` | `meaning` |
| S3 | `fill_blank` | `fill_blank` | `exampleSentence` |
| S4 | `sentence_build` | `sentence_build` | `exampleSentenceExtended` |
| S5 | `sentence_output` | `sentence_output` | `exampleSentenceExtended` |

接口和持久化内容校验必须同时允许 S1/S2 的两组合法组合；内容构建与覆盖检查必须依据词库版本选择其中一组，不能把任意阶段和任意题型放宽为合法。

## 5. 分步实施与验证

### 阶段 A：契约与迁移

实施：

1. 增加内容模型类型、三层语境字段和 additive D1 migration。
2. 历史行默认标记为 v1；新导入版本明确标记为 v2。
3. D1 与内存 repository 完整读写内容模型和三层语境。

验证：

- 从 0001 连续执行到最新 migration 成功。
- 迁移前种子数据迁移后仍为 v1，原字段无变化。
- v2 版本写入再读取后字段和值完全一致。

### 阶段 B：CSV 与管理端导入

实施：

1. 更新下载模板、严格表头、预览与提交结构。
2. 三个语境字段均校验非空和 2,000 字符上限。
3. 更新导入区字段说明和示例。

验证：

- 模板下载内容与导入器表头完全一致。
- 合法 RFC4180 引号、换行和 UTF-8 内容仍可解析。
- 缺少任一语境、旧表头、多列、错序或超长字段均返回明确行号与字段。
- 浏览器本地解析不产生网络请求。

### 阶段 C：内容构建与覆盖

实施：

1. 将阶段任务表改为按内容模型选择。
2. S0 使用短语，S3 使用基础句，S4/S5 使用扩展句。
3. S1/S2 在 v2 中调整为“先选择、后默写”。
4. 题目生成、编辑、批准和发布覆盖检查共享同一模型规则。

验证：

- v2 单词生成六题且阶段、题型、提示、答案逐项匹配冻结表。
- 基础句无法挖空目标词时阻止 S3 发布。
- 扩展句不含目标词或无法形成可见乱序时阻止 S4/S5 发布。
- v1 构建结果与改造前保持一致。

### 阶段 D：任务 API 与旧快照兼容

实施：

1. 学习任务与 exercise item schema 接受 v1/v2 的合法 S1/S2 配对。
2. CourseRuntime 继续从 immutable exercise item 复制任务快照。
3. 旧内容恢复器继续只修复旧 JSON 形态，不改写语义。

验证：

- v1 与 v2 六阶段任务均通过 API schema。
- 非法阶段/题型组合仍被拒绝。
- 已持久化的旧 exercise item 和 lesson task 可恢复、作答和生成报告。
- 新 lesson task 创建后即使词库行变化，提示与答案仍不变化。

### 阶段 E：学习端与视觉核验

实施：

1. 沿用六个现有任务渲染器，调整必要的阶段说明文案。
2. S0 明确展示短语语境；S4/S5 明确使用扩展句。
3. 更新视觉审阅稿或补充本次学习阶段审阅页面，覆盖桌面与手机宽度。

验证：

- 真实浏览器依次完成 S0-S5，观察到语境由短语到基础句再到扩展句。
- S1 为选择，S2 为输入，交互顺序无倒挂。
- 错答、反馈、S5 先写后揭示和结课流程无回归。
- 375px 与桌面宽度无横向溢出、遮挡或不可操作控件。
- 控制台无 error，关键请求无 4xx/5xx。

### 阶段 F：回归与交付

验证命令以 `package.json` 当前脚本为准，至少包含：

1. 定向单元与组件测试。
2. TypeScript 类型检查。
3. 全量单元/组件测试。
4. 生产构建。
5. Cloudflare 隔离检查。
6. stack E2E 与真实浏览器关键路径。
7. `git diff --check` 和最终代码审查。

## 6. 风险与防护

| 风险 | 防护 |
| --- | --- |
| 直接交换 S1/S2 导致旧快照解析失败 | 版本化内容模型，schema 只放宽两组合法配对 |
| 旧词库被迫补写新字段 | migration 默认 v1，新列只对 v2 导入必填 |
| 构建与覆盖检查采用不同映射 | 使用同一个按模型选择的阶段任务定义 |
| S0/S3/S4/S5 误用同一句 | 每个生成函数显式选择字段并用行为测试锁定 |
| 新版本导入后字段丢失 | repository round-trip 与 D1 migration 测试 |
| 队列优化发生回归 | 不改队列模块，并运行既有 queue 全量测试 |
| 已发布内容被重建 | 继续执行 draft-only 与 CAS 约束 |

## 7. 验收标准

满足以下全部条件才算完成：

1. 下载模板表头与冻结的六列完全一致。
2. 新导入缺少任一三层语境字段时无法提交，并指出具体行与字段。
3. 新版本数据库标识为 v2，三个语境字段可无损读回。
4. v2 的 S0-S5 映射、提示和答案与第 4.3 节完全一致。
5. v1 已发布版本和既有 lesson task 快照继续通过兼容测试。
6. 非法 S1/S2 配对不能绕过 schema 或发布覆盖检查。
7. 错题间隔、单词重复上限和 lesson queue 既有测试全部通过，相关代码无业务修改。
8. 管理端导入、构建、批准、发布与学习端 S0-S5 在真实 stack 上闭环通过。
9. 视觉审阅稿、真实渲染截图、响应式和控制台检查齐全。
10. 全量测试、构建、Cloudflare 检查和 `git diff --check` 全部通过。
