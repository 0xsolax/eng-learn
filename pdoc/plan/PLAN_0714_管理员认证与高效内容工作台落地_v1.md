## 2. 实施结论

本轮不是从零搭建管理端，也不是单纯替换 CSS。

当前仓库已经具备词库导入、版本构建、覆盖率、练习审核、发布、课程创建和学习码轮换等业务闭环。实施重点是先补齐配置型管理员登录，再把现有业务页面按已确认设计收口为高密度工作台。

实施顺序固定为：

1. 冻结认证契约和测试接缝。
2. 建立配置型管理员、D1 会话和持久冷却。
3. 调整 Worker 文档、API 和三种身份的路由边界。
4. 落地管理员登录、会话恢复、退出和真实身份显示。
5. 在不改业务规则的前提下调整四个现有管理页面。
6. 完成响应式、无障碍、敏感信息和真实 Worker/D1 验收。

不得先做页面美化再补认证。否则会形成可见但无法安全进入的登录页，并继续保留错误的会话失效行为。

## 3. 严谨目标

### 3.1 总目标

在不改变学生端、不重写现有内容与课程业务规则、不引入新 UI 框架的前提下，交付一个可通过终端初始化、可安全登录、可恢复会话、可明确退出，并符合已确认视觉方案的管理后台。

### 3.2 完成必须同时满足

1. 管理员使用终端初始化的账号和密码登录，不使用 10 位学习码。
2. 浏览器不持有管理员密码、密码校验值、ADMIN_API_TOKEN 或可重放的静态管理员凭证。
3. 管理员原始密码不写入 Cloudflare Secret；Secret 只保存版本化账号配置和不可逆密码校验值。
4. 应用会话使用随机令牌；浏览器只持有 Secure、HttpOnly、SameSite=Strict Cookie，D1 只保存令牌哈希。
5. 登录、退出和所有基于浏览器身份的管理写请求校验精确 Origin；service token 写调用按 API-only 凭证边界处理。
6. Cloudflare Access、配置型管理员会话和 service token 三种身份不互相降级。
7. x-admin-token 只能访问受控管理 API，不能加载管理端 HTML、建立浏览器会话或充当密码。
8. 未登录可以加载 /admin/login，但不能加载业务壳或调用业务数据 API。
9. 只有明确的会话缺失、过期、撤销或身份无效可以卸载业务壳；Origin 拒绝、业务冲突和普通服务错误保留当前页面。
10. 登录后直接进入 /admin/source-versions，或恢复经过路由白名单验证的站内管理路径。
11. 管理端只保留词库工作台和课程工作台两个真实导航入口。
12. 已有版本时列表优先；已有课程时课程表格优先；空状态才默认展开创建区。
13. 版本详情使用命令条、流水线、阻断摘要、单词 × S0 至 S5 矩阵和紧凑审批列表。
14. 练习详情具备桌面双栏审核、字段错误、未保存离开保护和已发布只读。
15. 一次性学习码支持复制结果反馈，关闭后不能从列表恢复；轮换确认位于触发对象附近。
16. 320 至 479px 的业务页不提供发布、丢弃、批量审批、轮换学习码等写操作；登录仍完整可用。
17. 类型、lint、单元、组件、API、D1、浏览器、构建、Cloudflare 检查和敏感产物扫描全部通过。

## 6. 管理员配置方案

### 6.1 单一 Secret

第一版使用一个版本化 Secret：ADMIN_AUTH_CONFIG。

Secret 包含：

| 字段 | 规则 |
| --- | --- |
| version | 固定为 1，不识别的版本 fail closed |
| username | 3 至 64 个允许字符，trim 后转小写作为登录标识 |
| displayName | 1 至 64 个可见字符，直接作为管理端显示名 |
| credentialId | 每次初始化或重置都重新生成的随机 ID |
| algorithm | 固定 PBKDF2-HMAC-SHA256 |
| iterations | 固定 600000 |
| salt | 16 字节加密安全随机值，base64url |
| verifier | 32 字节派生值，base64url |
| rateLimitKey | 32 字节随机密钥，用于 HMAC 客户端限流键 |

Secret 不包含原始密码。用户名和显示名虽然不是密码，也与校验值一起放入 Secret，避免生产配置散落于代码和普通变量。

ADMIN_AUTH_CONFIG 的唯一传输格式冻结为：

    v1.<base64url(canonical UTF-8 JSON)>

canonical JSON 使用固定字段顺序且不含换行；base64url 不带 padding。Worker 必须先校验 v1 前缀、base64url 和严格 JSON schema，任何重复字段、未知字段或未知版本都 fail closed。

### 6.2 终端初始化

新增两个明确命令，但远端命令只有在另行授权后执行：

| 命令 | 行为 |
| --- | --- |
| pnpm admin:init:local | 无回显读取账号、显示名、密码和二次确认；派生校验值；原子写入 Git 忽略的 .dev.vars；文件权限收紧为 0600 |
| pnpm admin:init:remote | 完成同样校验后，通过标准输入调用 Wrangler secret put；Secret 不出现在命令参数、终端正文或日志 |

初始化要求：

- 账号只允许字母、数字、点、下划线、短横线、加号和 @。
- 密码不 trim、不做静默截断，长度 15 至 128 个 Unicode code point；允许空格和 Unicode，不增加大小写、数字或符号组合规则。
- 初始化脚本拒绝与账号、显示名、项目名相同或命中内置常见弱密码阻断表的值；第一版不调用外部泄露密码服务。
- 两次密码不一致立即失败。
- 不提供仓库内默认账号或默认密码。
- 不把 Secret JSON 输出到 stdout。
- 重置账号或密码时生成新的 credentialId，使旧应用会话自动失效。
- 脚本失败时不得留下半写入 .dev.vars。
- 更新本地配置时只替换唯一 ADMIN_AUTH_CONFIG 行，保留其他变量、注释、顺序和结尾换行；发现重复定义时停止，不猜测哪一条有效。
- 临时文件与目标文件位于同一目录，写入完成后原子 rename；最终权限固定 0600。
- 整栈测试必须让 Wrangler 实际读取该格式，不能只用脚本自身解码证明正确。

### 6.3 密码校验参数与 Worker CPU 门禁

密码校验采用 PBKDF2-HMAC-SHA256，600000 次，16 字节随机 salt，32 字节派生值，使用 Cloudflare Web Crypto deriveBits 和 timingSafeEqual。

Secret 已配置时，合法长度范围内的错误账号和错误密码都执行一次完整 PBKDF2，再合并账号与密码比较结果，避免用明显耗时差区分账号是否存在。账号比较使用规范化后的定长摘要；错误响应保持一致。

该参数符合 OWASP 当前 PBKDF2-HMAC-SHA256 推荐下限。Cloudflare Workers Web Crypto 支持 PBKDF2，但 Workers Free 当前每次请求 CPU 上限为 10ms；认证类工作通常可能超过该预算。

因此增加 G-CPU 门禁：

1. 不允许为了通过 Free 计划而静默降低迭代次数。
2. 本地实现必须记录 100 次成功和失败校验的 P50、P95。
3. 远端部署前必须在不记录凭证的诊断入口或一次性受控 smoke 中验证目标 Worker 没有 1102 CPU 超限。
4. P95 必须低于 1 秒，且目标计划的 CPU 上限必须覆盖实测值并保留余量。
5. 如果目标仍是 Workers Free 且无法通过，停止远端发布，由 Solazhu 在升级 Workers Paid、恢复 Cloudflare Access 或另行批准测试专用弱化方案之间选择。

不得把未达安全下限的快速哈希包装成“已经安全解决”。

### 6.4 精确站点 Origin

APP_ORIGIN 继续作为非 Secret 的唯一站点 Origin，不从请求 Host、Origin 或 Forwarded Header 动态推导。

- 本地整栈由测试 runner 注入本次独占 HTTPS Origin。
- 暂不配置自定义域名时，远端部署必须把 APP_ORIGIN 明确设置为实际 workers.dev Origin；当前报告记录的是 https://eng-learn.0xsolax.workers.dev，部署前仍需以实时 Worker URL 复核。
- APP_ORIGIN 缺失、非法或与请求 Origin 不完全一致时，登录、退出和管理写请求全部 fail closed。
- APP_ORIGIN 变更属于部署配置变更，仍需单独授权；本计划不执行远端修改。

### 6.5 浏览器认证部署模式

浏览器认证不能把 Cloudflare Access 和应用账号登录同时当作同一路径上的可选前门。Access 在请求到达 Worker 前执行；如果它保护整个 /admin*，未登录用户看不到应用登录页，应用 Cookie 也不能替代 Access Cookie。

新增非 Secret 配置 ADMIN_BROWSER_AUTH_MODE，只允许：

| 模式 | 当前用途 | 浏览器身份 | /admin/login | 退出 |
| --- | --- | --- | --- | --- |
| application_session | 当前无自定义域名调试 | application_session | Worker 未认证公开 | 调用应用 logout API，撤销 D1 会话 |
| cloudflare_access | 未来 Access 部署 | cloudflare_access | 由 Access 前门处理；应用不提供账号密码表单 | 导航到 /cdn-cgi/access/logout |

共同规则：

- service_token 始终只用于受控 API，不属于浏览器模式。
- 当前实施默认 application_session；不得在同一部署上再用 Access 策略拦截全部 /admin*。
- 切换 cloudflare_access 前必须单独修改并验证 Access application/policy；这不在本轮远端授权内。
- cloudflare_access 退出会清除 Access 授权 Cookie，但 Cloudflare 文档说明已签发 token 完全停止接受可能需要 20 至 30 秒；界面不得宣称应用已经同步撤销 D1 会话。

Access 退出依据：https://developers.cloudflare.com/cloudflare-one/access-controls/access-settings/session-management/

## 7. 应用会话与登录冷却

### 7.1 会话模型

新增 additive migration 0010_add_admin_sessions.sql，建立：

admin_sessions：

- id：随机 UUID。
- token_hash：SHA-256 哈希，唯一；不保存 raw token。
- credential_id：创建会话时的配置版本标识。
- created_at：服务端时间。
- expires_at：绝对过期时间。
- revoked_at：退出时写入；未退出为 null。

admin_login_rate_limits：

- key_hash：HMAC 后的客户端限流键，唯一；不保存原始 IP。
- window_started_at：当前失败窗口开始时间。
- failure_count：窗口内失败数。
- blocked_until：冷却截止时间。
- updated_at：最后更新时间。

### 7.2 Cookie

| 项目 | 冻结值 |
| --- | --- |
| 名称 | __Host-eng_learn_admin_session |
| 值 | 32 字节随机 opaque token 的十六进制 |
| Path | / |
| HttpOnly | 是 |
| Secure | 是 |
| SameSite | Strict |
| Max-Age | 28800 秒 |
| 会话时长 | 8 小时绝对有效期，不滑动延长 |

本地整栈使用 HTTPS。不得为了普通 HTTP 调试去掉 Secure。

### 7.3 会话判定

服务端按以下顺序判定应用会话：

1. Cookie 不存在或格式非法：admin_session_required。
2. token hash 不存在：admin_session_required。
3. revoked_at 已存在：admin_session_revoked。
4. expires_at 不晚于服务端当前时间：admin_session_expired。
5. credential_id 与当前 Secret 不同：admin_session_revoked。
6. 其余返回 application_session 身份。

退出只撤销当前会话。更新 ADMIN_AUTH_CONFIG 时 credentialId 改变，所有旧会话在下一次请求时自动失效。

### 7.4 持久冷却

第一版采用 D1 持久冷却，不使用 Worker isolate 内存计数。

冻结规则：

- 客户端键使用 rateLimitKey 对 CF-Connecting-IP 做 HMAC；本地测试使用受控 fixture 键。
- 15 分钟窗口内最多允许 5 次失败。
- 每次合法形状的登录请求在 PBKDF2 前通过单条 D1 原子语句占用一个尝试名额；同一窗口只允许前 5 个名额进入密码校验。
- 第 5 个名额校验失败后开始 15 分钟冷却，并返回 admin_login_rate_limited；第 6 个及后续并发请求不进入 PBKDF2。
- 冷却响应包含 retryAfterSeconds，HTTP 同时返回 Retry-After。
- 冷却期内在密码校验前拒绝，避免持续消耗 PBKDF2 CPU。
- 登录成功清除当前客户端键的失败状态。
- 账号不存在和密码错误统一返回 invalid_admin_credentials。
- 未配置 Secret 返回 admin_not_configured，不伪装为密码错误。
- 冷却过期后重建窗口，不永久锁定唯一管理员。
- 写入必须使用单条原子 UPSERT/RETURNING 或 D1 事务，禁止 read-modify-write 竞争。
- 20 个同客户端并发错误请求中，最多 5 个返回凭证错误，其余必须返回冷却；不得让 20 个都进入密码校验。
- 过期会话和旧限流行只做机会式清理；清理失败不能改变本次登录真假结果。

该规则防止单一来源暴力尝试，但不能替代 Cloudflare Access 或全局 WAF 对分布式攻击的防护。第一版不增加 CAPTCHA。

D1 事务依据：https://developers.cloudflare.com/d1/worker-api/d1-database/

## 8. 三种管理员身份与路由边界

### 8.1 身份来源

AdminIdentity.source 扩展为：

- cloudflare_access
- application_session
- service_token

管理员会话 DTO 固定包含：

- id
- source
- displayName
- email 可选

displayName 只由服务端确定：application_session 使用 Secret 中的 displayName；cloudflare_access 优先使用已验证 email，否则使用 subject；service_token 使用固定运维显示名，不接受请求传入。

### 8.2 防降级顺序

| 条件 | 唯一允许的验证方式 | 失败行为 |
| --- | --- | --- |
| 存在 Cloudflare Access assertion | 只验证 Access | 无效时不回退 Cookie 或 service token |
| application_session 模式、无 Access、存在管理员 Cookie | 只验证应用会话 | 无效时不回退 service token |
| 无 Access、无 Cookie、请求为 /api/admin/* 且存在 x-admin-token | 只验证 service token | 无效返回身份错误 |
| 管理端 HTML 文档携带 x-admin-token | 不接受 service token | 返回 401/403，不加载 HTML |
| 三者都没有 | 不认证 | 文档进入登录页或 API 返回 JSON 401 |

### 8.3 Worker 路由顺序

routeRequest 必须按以下顺序处理：

1. GET 或 HEAD /admin/login：application_session 模式下未认证可加载 SPA 静态资源；cloudflare_access 模式由 Access 前门处理。
2. POST /api/admin/auth/login：仅 application_session 模式开放；校验 Origin、配置、冷却和凭证。cloudflare_access 模式返回 404，不建立应用 Cookie。
3. POST /api/admin/auth/logout：仅 application_session 模式开放；校验 Origin；Cookie 缺失、过期或撤销时仍幂等清除 Cookie。cloudflare_access UI 不调用该接口。
4. 其他 /admin 文档：按 ADMIN_BROWSER_AUTH_MODE 只接受当前浏览器身份；不把另一模式当作降级备选。
5. GET /api/admin/session：接受三种身份并返回统一 DTO。
6. 其他 /api/admin/*：经过统一管理 API 鉴权。
7. /api/app/*：保持学习者会话边界，不复用管理员身份。

未认证管理 API 始终返回 JSON 错误，不返回登录 HTML。

application_session 整栈测试必须关闭当前 server/e2e.ts 对 /admin* 的 Access assertion 自动注入，改用临时 ADMIN_AUTH_CONFIG、真实登录 Cookie 和全新 D1。Access 验证保留为独立单元/API 场景，不能掩盖应用会话失败。

## 9. 认证 API 与错误契约

### 9.1 API

| 方法与路径 | 请求 | 成功 |
| --- | --- | --- |
| POST /api/admin/auth/login | username、password | 200 + AdminSession DTO + Set-Cookie |
| POST /api/admin/auth/logout | 无 body | 200 + loggedOut=true + 清 Cookie |
| GET /api/admin/session | 无 | 200 + AdminSession DTO |

登录请求字段在共享 Zod schema 中限制长度；密码不会出现在成功 DTO、错误 details、日志或 operation ledger。

登录请求体上限固定 4 KiB。无论 Content-Length 是否存在，都必须在 JSON 解析前按实际读取字节数限流；超限返回 413，不进入 D1 或 PBKDF2。

### 9.2 稳定错误

| code | HTTP | 是否卸载业务壳 | UI |
| --- | ---: | --- | --- |
| invalid_admin_credentials | 401 | 否 | 保留账号、清空密码、焦点回密码 |
| admin_login_rate_limited | 429 | 否 | 禁止提交并显示 retryAfterSeconds |
| admin_not_configured | 503 | 否 | 提示先完成终端初始化 |
| admin_session_required | 401 | 是 | replace 到登录页 |
| admin_session_expired | 401 | 是 | replace 到登录页并显示已过期 |
| admin_session_revoked | 401 | 是 | replace 到登录页并显示已失效 |
| admin_identity_invalid | 401 | 是 | 重新登录，不降级其他身份 |
| origin_forbidden | 403 | 否 | 保留当前页面，提示刷新或重试 |
| dependency_failure | 503 | 否 | 登录服务或 D1 暂不可用 |
| internal_error | 500 | 否 | 不猜测认证结果 |

adminAuthorizationBoundary 不再按 HTTP 状态广播，而只按明确会话失效 code 广播。非法非 JSON 401/403 视为 fail closed，但只在会话检查入口处理，不让普通业务请求凭状态码误卸载页面。

### 9.3 身份 × 方法 × Origin

| 身份/入口 | GET/HEAD | POST/PUT/PATCH/DELETE |
| --- | --- | --- |
| 登录与退出入口 | 不适用 | 必须精确匹配 APP_ORIGIN |
| application_session | 不要求 Origin | 必须精确匹配 APP_ORIGIN |
| cloudflare_access | 不要求 Origin | 必须精确匹配 APP_ORIGIN |
| service_token | API 读取允许；HTML 禁止 | API 写允许且不依赖 Origin；HTML 禁止 |
| 未认证 | 登录文档按模式处理；业务 API JSON 401 | JSON 401，不返回 HTML |

service token 不使用环境 Cookie，不受浏览器 CSRF 威胁；要求可伪造 Origin 不会增加它的鉴权强度，反而会破坏现有受控非浏览器调用。因此本计划把已确认设计中的“所有管理写请求”澄清为“所有基于浏览器身份的管理写请求”，并相应修改现有身份矩阵测试。

### 9.4 缓存与受保护文档跳转

- /admin/login、其他 /admin HTML、登录/退出/session 和全部管理 API 响应统一 Cache-Control: no-store。
- application_session 模式下，未认证 GET /admin 或任意受保护深层文档返回 302，Location 为 /admin/login?returnTo=<编码后的站内 path+query>。
- 未认证 HEAD 使用同一 302 和 Location，不返回 HTML body。
- returnTo 只取当前 URL 的 path+query，不接受 scheme、host 或 fragment。
- Cookie 过期、撤销或非法时，文档 302 同时发送清 Cookie Header。
- /api/admin/* 未认证继续返回 no-store JSON 401，绝不 302。
- SPA 内部导航由 router/AdminGate 执行等价 replace；浏览器 BFCache 恢复或 pageshow persisted 时必须重新验证会话。

## 10. 前端认证与应用壳

### 10.1 路由

- /admin/login 是 AdminGate 的同级公开路由。
- 受保护路由使用明确 meta 标记。
- 未登录进入受保护路由时，用 replace 转到登录页。
- returnTo 只接受路由表中已注册且标记为受保护的管理路由。
- 明确拒绝 /admin/login、外部 URL、协议相对 URL、未知路由和循环路径。
- 空值和非法值统一回到 /admin/source-versions。

### 10.2 登录页状态机

登录页实现：

- checking：检查既有会话，不挂载可编辑表单。
- idle：账号和密码可编辑。
- submitting：禁用重复提交。
- invalidCredentials：保留账号、清空密码、焦点回密码。
- coolingDown：按服务端 retryAfterSeconds 显示剩余时间；到期后重新允许提交。
- notConfigured：提示终端初始化，不显示默认凭证。
- networkError：保留账号并允许重试。
- serviceError：不猜测是否登录成功。
- expired：显示会话过期后重新登录。
- loggedOut：显示已安全退出。

账号 autocomplete=username，密码 autocomplete=current-password。错误区域紧邻表单并使用 role=alert。

### 10.3 应用壳固定规格

- 1280px 以上侧栏 224px；导航项 40px 高，Lucide 图标 18px。
- 主工作区最大宽度 1440px，页面 padding 24px。
- 保留“跳到主要内容”，并验证焦点落到 admin-main。
- 页面标题 24px；标题区最多一个主动作和两个次动作。
- 顶部上下文固定映射：
  - 词库列表：词库工作台 / 词库版本。
  - 版本详情：词库工作台 / {词库名} / v{版本号}。
  - 练习详情：{词库名} / v{版本号} / {单词} / {阶段}。
  - 课程页：课程工作台 / 学习者与课程。
- 上下文中的动态值来自子页面已经获得的服务端 DTO，不由路由参数猜名称。

### 10.4 会话上下文

不为本轮引入大型全局 Store。使用一个 typed provide/inject 会话上下文向 AdminLayout 提供：

- 当前 AdminSession。
- refreshSession。
- logout。
- clearPrivateState。

另提供 typed page-context 接缝，由已经读取业务数据的子页面上报面包屑、页面标题和可选 dirty 离开守卫；AdminLayout 不为顶部上下文重复请求同一业务 API。页面卸载时清空该上下文，防止旧对象名称串到下一路由。

AdminGate 只有在 getAdminSession 成功后才挂载业务壳。会话失效时先卸载私有子树，再 replace 到登录页。

### 10.5 退出

- 完整侧栏底部显示 displayName、内容管理员和退出。
- 72px 侧栏与移动布局把同一身份和退出迁移到顶部栏，两处不同时显示。
- 退出中禁用重复点击。
- application_session 成功：撤销 D1 当前会话、清 Cookie、清理会话上下文和页面子树，replace 到 /admin/login?reason=logged_out。
- cloudflare_access：不调用应用 logout API，导航到同源 /cdn-cgi/access/logout；service_token 不进入浏览器应用壳。
- 失败：不假装成功，不清空工作台，提示会话仍可能有效。
- 退出后浏览器后退再次进入受保护路由时，AdminGate 必须重新向服务端确认。
- 若当前页面存在 dirty 离开守卫，点击侧栏或顶部栏退出必须先经过同一确认；用户取消时不发送 logout 请求、不清空状态。

### 10.6 恢复矩阵

| 触发 | 保留/清除 | 用户动作与下一请求 | 焦点 |
| --- | --- | --- | --- |
| 有效会话访问 /admin/login | 不挂载登录表单 | getAdminSession 成功后 replace 安全 returnTo 或词库页 | 目标页主标题 |
| 登录 POST 网络结果未知 | 保留账号，清空密码，不假定成功 | 先调用 getAdminSession；成功进入工作台，未建立会话才允许重新输入 | 密码或目标页主标题 |
| 初次 session 检查网络失败 | 不挂载业务壳，不显示 Empty | “重新检查”再次 getAdminSession | 重新检查按钮 |
| 业务页 session 过期/撤销 | 卸载全部私有子树，只保留白名单 returnTo | replace 登录；重新登录后重新请求页面 DTO | 密码字段 |
| 普通业务网络失败 | 不把旧 DTO 标记为当前成功态 | 显示稳定错误区；用户主动重新读取对应 GET | 重试按钮 |
| 页面重新读取中 | 保持 Loading，不提前显示 Empty | 等服务端结果后进入 success/empty/error 唯一状态 | 不抢焦点 |
| 退出失败 | 保留当前会话、页面和 dirty 状态 | 允许重试 logout | 退出按钮或错误区 |
| 退出成功后后退/BFCache | 私有子树保持清除 | pageshow/路由重新 getAdminSession，失败回登录 | 登录标题 |

重新登录只恢复经过白名单验证的路由位置，不恢复旧页面内存、旧表单、旧 coverage 或旧课程列表。

## 13. 待确认的 TDD 测试接缝

进入任何测试或生产代码前，Solazhu 需要确认以下公共接缝：

1. 共享契约：Zod request/response/error schema。
2. 纯认证规则：配置解析、密码校验、Cookie 解析和身份优先级的公开函数。
3. 数据一致性：AdminSessionRepository、AdminLoginRateLimitRepository 与全新本地 D1。

5. HTTP 契约：Worker Request/Response、Set-Cookie、Retry-After、Origin 和状态码。
6. API client：adminApi 的 loginAdmin、logoutAdmin、getAdminSession。
7. Vue 行为：组件公开 props、事件、可访问 DOM、路由 meta 和可见状态。
8. 用户闭环：Playwright 浏览器与隔离 Worker/D1。
9. 视觉一致性：同一 fixture、同一视口下，视觉审阅稿与生产页面截图并排比较。

禁止：

- 测试私有函数或内部调用次数。
- 通过 CSS selector 证明服务端安全。
- 用 mocked UI E2E 宣称真实登录闭环完成。
- 用内存 repository 代替 D1 并发、唯一约束和持久冷却测试。
- 用 HTTP 401/403 状态代替稳定错误码判断会话失效。
- 在测试 fixture、trace、HAR、截图、console 或失败产物长期保存密码、Cookie、ADMIN_API_TOKEN 或学习码。

## 18. 风险、停止条件与回滚

| 风险 | 控制 | 停止条件 |
| --- | --- | --- |
| Free Worker CPU 无法承受 PBKDF2 | G-CPU 实测，不降低参数 | 1102、P95 超标或无资源余量 |
| Access 与应用登录互相挡住 | ADMIN_BROWSER_AUTH_MODE 互斥部署 | Access 拦截应用登录或应用 Cookie 无法进入业务路由 |
| Access 退出被误写成应用退出 | 按 source 分流 logout | Access 退出后立即重新认证却显示安全退出 |
| 登录页仍被统一鉴权拦截 | 先写 Worker 路由红测 | 未认证 GET/HEAD 不可达 |
| 身份降级 | assertion/cookie/token 优先级矩阵 | 任一无效高优先身份回退成功 |
| service token 加载 HTML | 文档与 API 分离 | x-admin-token 能获得管理 HTML |
| 冷却并发丢计数 | D1 原子 UPSERT/事务 | 并发测试计数错误 |
| 并发请求先穿过冷却再耗尽 CPU | PBKDF2 前原子占用名额 | 20 个并发请求中超过 5 个进入校验 |
| 会话撤销后仍可重放 | token hash + revoked_at + E2E | 旧 Cookie 仍成功 |
| 错误 403 误卸载工作台 | 只按稳定 code 广播 | origin_forbidden 导致私有状态丢失 |
| 页面重排破坏现有恢复 | 保留现有 operation token 测试 | 结果未知只能盲目重放 |
| 未保存编辑丢失 | dirty + route/beforeunload | 直接离开无确认 |
| 学习码进入产物 | fixture 脱敏 + 扫描 | 任何产物命中明文 |
| 新管理员 Secret 未被扫描 | 随机 canary + 扫描器自测 | 扫描器漏掉 ADMIN_AUTH_CONFIG、管理员 Cookie、verifier 或 rateLimitKey |
| E2E 误连生产 D1 | 临时根、独占端口、sentinel | 无法证明 D1 身份 |

回滚：

- migration 只 additive，不删除旧表或字段。
- 新认证功能按切片保持可独立回滚。
- 若应用会话实现失败，恢复到现有 Access/service token 代码路径，但不得把浏览器静态 token 作为替代。
- 本轮不执行远端 Secret、migration 或部署，因此本地计划阶段没有生产回滚动作。
- 后续远端发布前单独记录上一 Worker version、D1 migration 状态和 Secret 轮换步骤。

进入测试或编码前，需要 Solazhu 明确确认：

5. 如果 Codex 内置浏览器仍不可用，授权使用仓库现有 Playwright 测试脚本完成本地浏览器与视觉验收。
6. 远端 Secret 写入、远端 migration 和部署仍保持未授权，后续单独确认。
